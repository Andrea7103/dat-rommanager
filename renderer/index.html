DAT//ROMMANAGER
v0.9.0-beta · Universal DAT Management Toolkit
DAT//ROMMANAGER is an all-in-one toolkit for managing DAT files used in ROM preservation workflows. Designed to work alongside RomVault, ClrMamePro and RomCenter.
WHAT ARE DAT FILES?
A DAT file is a database describing a set of ROMs — their names, sizes and checksums (CRC32, MD5, SHA1). ROM managers use DAT files to verify your collection and identify missing or corrupt files.
MODULES
  • 01 — DAT Merger Combine hundreds of DAT files into one file for RomVault.
  • 02 — DAT Explorer Inspect any DAT: browse entries, view ROM checksums, see platform info.
  • 03 — DAT Splitter Split a DAT into multiple files by region, language, year, alphabet, type and more.
  • 06 — Checksum Calc Compute CRC32, MD5, SHA1, SHA256 for any file and verify against known hashes.
  • 04 — DAT Diff Compare two or more DAT files by checksum — find exclusive and common entries, then export exactly what you need.
This is a beta release. Modules 04–06 are in development. Check the Changelog after each update.
COMPATIBILITY
  • Windows 10 / 11 (x64)
  • macOS 12+ (Intel & Apple Silicon)
  • Linux (AppImage)
MODULE 01 // DAT MERGER
Status: READY · Stable
Combines multiple DAT files from a folder (and all subfolders) into a single output DAT. Supports both XML Logiqx and ClrMamePro formats.
HOW TO USE
  • Click the drop area or drag a folder onto it
  • The app scans recursively for all .dat and .xml files
  • Configure output settings on the right panel, then click ▶ MERGE ALL
  • A summary dialog shows full statistics after merge completes
CONFIG OPTIONS
  • Output filename Name of the generated .dat file
  • Version / Author Written into the output DAT header
  • Keep duplicates When OFF, identical entry names are deduplicated
  • Source prefix Prepends source DAT filename to each entry name
Very large merges (10,000+ entries) may take several seconds. The progress overlay shows real-time status.
MODULE 02 // DAT EXPLORER
Status: READY · Stable
Load and inspect any single DAT file. Browse all entries, view ROM checksums and see system information for the platform the DAT covers.
LAYOUT
  • Left panel Drop zone + DAT header info and statistics
  • Center top Scrollable sortable list of game entries
  • Center bottom ROM files for the selected entry — full names and all checksums. Drag the splitter bar to resize.
  • Right panel System Info: platform description, manufacturer, release year
NAVIGATION
  • Click any entry to view its ROM files
  • Use ↑ ↓ arrow keys to navigate with keyboard
  • Type in the search box to filter entries in real time
  • Sort buttons: A→Z, Z→A, ROMs↓ (most ROMs first), ROMs↑ (fewest first)
System Info describes the platform of the entire DAT, not the selected entry. It updates only when a new DAT is loaded.
MODULE 06 // CHECKSUM CALCULATOR
Status: READY · Stable
Computes cryptographic and integrity hashes for any file. Supports CRC32, MD5, SHA1 and SHA256.
HOW TO USE
  • Drop files onto the zone or click to open the file picker
  • Multiple files can be processed at once
  • Dropping new files clears previous results automatically
  • Toggle algorithms on/off before processing
  • Click COPY next to any hash to copy to clipboard
VERIFY FIELD
Each result card has a VERIFY input at the bottom. Paste any expected hash (from a DAT file, a website, etc.) and the app instantly tells you if it matches any computed hash, regardless of algorithm.
ALGORITHMS
  • CRC32 Fast 32-bit checksum. Standard in ROM DAT files.
  • MD5 128-bit hash. Common in older DAT formats.
  • SHA1 160-bit hash. Standard in No-Intro and Redump DATs.
  • SHA256 256-bit hash. Modern high-security standard.
CRC32 is the most widely used checksum in ROM preservation. For basic verification against a DAT file, CRC32 alone is usually sufficient.
MODULE 03 // DAT SPLITTER
Status: READY · Stable
The DAT Splitter divides a single DAT file into multiple output files according to a chosen criterion. Each output is a valid XML Logiqx DAT with its own header, ready for RomVault, ClrMamePro or RomCenter.
HOW TO USE
  • Drop a DAT file onto the left panel, or click to browse
  • Or click USE DAT FROM EXPLORER to reuse a DAT already loaded in Module 02 — no re-parsing needed
  • Choose a split mode from the center panel
  • Review the preview list — each group shows label, filename and entry count
  • Uncheck any groups you don't want to save, then click ▶ SPLIT & SAVE
SPLIT MODES
  • Region Detects (USA), (Europe), (Japan), (World) etc. in entry names. Also reads the rom.region attribute. No tag → Unknown / No Region Tag.
  • Language Detects language codes: (En), (It), (Fr), (De), (Ja) etc. Multi-language entries like (En,Fr,De) are grouped together.
  • Year One file per release year. If year field is empty, extracts 4-digit year from entry name.
  • Decade Groups by decade: 1980s, 1990s, 2000s, 2010s…
  • Alphabetical Five buckets: A-E · F-J · K-O · P-T · U-Z · # (non-letter).
  • Manufacturer Splits by the manufacturer/developer field in the DAT.
  • Category Splits by category field. Useful for MAME and TOSEC DATs with category metadata.
  • Type Three groups: BIOS / System (isbios="yes" or [BIOS] tag) · Clones (entries with cloneof/romof) · Parents / Originals.
  • Release Status Detects tags: Beta · Proto · Demo · Sample · Unlicensed · Hack · Bootleg · Pirate · Released.
  • ROM Count Groups by number of ROM files per entry: 1 · 2-5 · 6-20 · 21-100 · 100+.
OUTPUT FILES
Each output file is a valid XML Logiqx DAT with updated header. Filename is auto-generated as OriginalName - GroupLabel.dat. Invalid filename characters are replaced with underscores.
Use USE DAT FROM EXPLORER when working with large DATs (80MB+) — avoids parsing the file twice.
Region and Language detection follows No-Intro, Redump and TOSEC naming conventions. DATs with non-standard naming may produce unexpected groups — always review the preview before saving.
MODULE 04 // DAT DIFF
Status: BETA · In active development
The DAT Diff module compares two or more DAT files and identifies which entries are unique to each file, and which are shared. Comparison is based on CRC32 and SHA1 checksums — not on entry names, which may differ between sources.
COMPARISON MODES
  • A vs B Direct comparison between two DAT files. Shows what is exclusive to A, exclusive to B, and what they have in common.
  • Master + N Compare a master DAT against one or more secondary DATs. Useful for checking what a HBMAME or extended set adds over a standard MAME DAT, for example.
HOW TO USE
  • Choose a comparison mode from the tabs at the top of the left panel
  • Drop a DAT file into each slot, or click to browse
  • In Master + N mode, additional secondary slots appear automatically as you load files
  • Click ⊜ COMPARE to run the analysis
  • Browse results using the three tabs: Only in A, Only in B, Common
  • Use the search box and sort button to navigate large result sets
  • Use the checkboxes to select specific entries you want to export
EXPORT OPTIONS
  • What to export Choose any combination of the three categories (Only A, Only B, Common)
  • Separate files One output DAT per selected category
  • Merged into one All selected categories combined into a single DAT
  • All entries Export every entry in the selected categories
  • Checked entries only Export only the entries individually checked in the viewer
COMPARISON METHOD
Two entries are considered identical if their ROM file set produces the same fingerprint — a sorted combination of CRC32 and SHA1 values. This correctly handles cases where entry names differ between sources (e.g. TOSEC vs No-Intro naming conventions).
Typical use case: load a standard MAME DAT as Master and HBMAME as Secondary. "Only in Secondary" will give you exactly the homebrew-exclusive entries.
Entries with no ROM files (header-only entries) have an empty fingerprint and will always be grouped together as "common" or may produce unexpected results. Review before exporting.
DAT // FILE FORMATS
Reference guide for supported formats
XML LOGIQX
Standard format used by No-Intro, Redump and modern TOSEC releases.
<datafile> <header> <name>Nintendo - Game Boy Advance</name> </header> <game name="Game Title (USA)"> <rom name="Game Title (USA).gba" size="4194304" crc="AABBCCDD" md5="..." sha1="..."/> </game> </datafile>
CLRMAMEPRO FORMAT
Older text-based format used by ClrMamePro and some MAME tools.
clrmamepro ( name "My Collection" version "1.0" ) game ( name "Game Title" rom ( name "Game Title.gba" size 4194304 crc AABBCCDD ) )
ClrMamePro DATs are auto-converted to XML Logiqx during merge. Original files are never modified.
KNOWN DAT SOURCES
  • No-Intro Cartridge-based systems — format: Manufacturer - System
  • Redump Disc-based systems — format: Manufacturer - System
  • TOSEC Broad coverage — format: System - Category (Region)
  • MAME / HBMAME Arcade machines and homebrews
